Программа поиска фрагмента.
// Замечания:
1. При работе ПГП обратить внимание на вычисление псевдоградиента и оценок - правильные ли массивы берутся

Описание работы программы:
1. Программа получает на вход исходные данные: файл исходного изображения, файл фрагмента.
2. Предварительная обработка двух кадров исходных данных(исходного изображения и фрагмента):
   Кадр исходных данных включает 
   - данные: 
		1) Массив значений яркости (в градациях серого);
		2) Размеры кадра по вертикали и горизонтали;
		3) Координаты центра кадра;
		4) Объект изображения
   - методы:
		1) Преобразование объекта изображения в массив яркостей;
		2) Загрузка данных из графического файла в объект изображения;
		3) Преобразование массива яркостей в объект изображения;
		4) Вычисление координат центра изображения и его размеров.
3. Программа получает данные для запуска алгоритма поиска:
	- тип функции приоритета;
	- количество итераций;
	- объем локальной выборки;
4. Обработка входных данных - собственно локализация
   4.1. Создание сетки ПГП
   4.2. Инициализация сетки
   4.3. Выявление V-ПГП
5. Вывод результатов локализации
   5.1. Отрисовка границ фрагмента на исходном изображении;
   5.2. Сбор данных о значениях функции приоритета и количестве итераций
   5.3. Вывод графиков распределений ФП и количества итераций, а также сходимости оценок

Структура программы:

1. Подсистема загрузки исходных данных - изображения и фрагмента

2. Подсистема поиска и локализации фрагмента

   2.1. Модуль создания сетки ПГП // GridMaker
		2.1.1. Получение входных данных: // в конструкторе
		- Кадры изображения и фрагмента;
		- Объем локальной выборки, тип ФП;
		2.1.2. Вычисление размеров сетки; // getGridSize
		2.1.3. Инициализация ПГП в каждой ячейке сетки: // initGrid
		- Заполнение массива значений яркости;
		- Определение ОЛВ и типа ФП;
		2.1.4. Формирование выходных данных: 
		- Сетка инициализированных ПГП.

   2.2. Модуль поиска V-ПГП
		2.2.1. Получение входных данных:
		- Сетка ПГП;
		- Общее количество шагов алгоритма поиска;
		2.2.2. Первичный обход сетки ПГП:
		- Заполнение локальной выборки;
		- Выполнение итерации оценивания;
		- Вычисление значения функции приоритета;
		2.2.3. Определение V-ПГП:
		- Повторение в течение заданного количества шагов алгоритма:
			- Поиск индексов ПГП с наилучшим значением ФП;
			- Заполнение локальной выборки одной выбранной процедуры
			- Выполнение итерации оценивания одной выбранной процедурой;
			- Вычисление Функции Приоритета одной выбранной процедуры;
		- Определение ПГП с наибольшим количеством сделанных итераций;

	2.3. Модуль получения параметров алгоритма
		 - число шагов алгоритма;
		 - объем локальной выборки;
		 - тип функции приоритета.
		 
	2.4. Модуль отдельной ПГП
		 - Массив значений яркости;
		 - Интерполяция билинейная;
		 - Локальная выборка;
		 - Счетчик итераций;
		 - Значение ФП;
		 - Векторы оценок параметров;
		 - Выполнение итерации:
			- Заполнение локальной выборки;
			- Оценивание параметров;
			- Вычисление ФП;

	2.5. Локальная выборка
	     - координаты точек на исходном изображении
		 - координаты точек на эталонном фрагменте
		 - значения яркости в точках на исходном изображении
		 - значения яркости в точках на эталонном фрагменте

3. Подсистема вывода результатов поиска и локализации
	3.1. Модуль сбора выходных данных: OutDataCollector
		Входные данные: объект типа Locator;
		Выходные данные: 
		 - Объект V-ПГП(VeritasData):
		   - глобальные координаты углов-вершин фрагмента - прямоугольника;
		   - массивы значений векторов оценок параметров;
		   - значение ФП и количество итераций;
		 - Сетка ПГП(GridStatistics):
		   - массивы значений ФП и количества итераций для всех ПГП;
		   - те же массивы, но отранжированные по убыванию.
